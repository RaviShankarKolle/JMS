Implemented functionality
1Create book — POST /api/v1/books
Stores title, author, genre, ISBN, total copies, and optional description. New books start as AVAILABLE with available_copies equal to total_copies.
Validation: non-blank fields, ISBN pattern (10–17 digits/hyphens), at least one copy.

Update book — PUT /api/v1/books/{bookId}
Updates catalog metadata (same fields as create). Returns updated book payload.

Soft remove book — DELETE /api/v1/books/{bookId}
Marks the book as REMOVED (not physically deleted from the database).

List books (paginated) — GET /api/v1/books?page=&size=
Returns only books with status AVAILABLE, newest first. Page defaults to 0, size to 10; size is capped between 1 and 100. Response includes total count for pagination.

Reserve a copy — POST /api/v1/books/{bookId}/reserve
Decrements available_copies and updates status (AVAILABLE or RESERVED when no copies left). Fails with conflict if no copies are available or book is removed.

Issue — POST /api/v1/books/{bookId}/issue
Adjusts status for borrow workflow (AVAILABLE vs BORROWED depending on remaining copies). Intended for service-to-service use (e.g. borrow-service).

Return — POST /api/v1/books/{bookId}/return
Increments available_copies and sets status back to AVAILABLE.

ISBN uniqueness — Creating a book with an ISBN that already exists returns a conflict-style business error.

Domain events — On add, update, and remove, the service publishes internal BookCatalogEvent instances (BOOK_ADDED, BOOK_UPDATED, BOOK_REMOVED). BookEventListener handles these asynchronously and logs them (hook for future Kafka or other integrations).

Consistent API envelope — Success and error responses use a shared ApiResponse shape (status, data, optional error body, metadata, timestamp).

Global error handling — Application exceptions map to appropriate HTTP statuses via a central exception handler.

Security
JWT — Stateless API secured with a Bearer JWT filter; roles are read from the token.
Rules (summary):
Swagger/OpenAPI, Swagger UI, and GET /actuator/health are public.
POST reserve/issue/return are permitted without JWT so other microservices (e.g. borrow-service via Feign) can call them without a user token.
GET /api/v1/books (list) requires an authenticated user.
Create, update, and delete require LIBRARIAN or ADMIN.
JSON responses for 401 (missing/invalid JWT) and 403 (insufficient role).

        eventPublisher.publishEvent(new BorrowDomainEvent(type, borrowId, userId, bookId, Instant.now()));
        log.info("Borrow event: type={}, borrowId={}, userId={}, bookId={}", type, borrowId, userId, bookId);
        if ("BORROW_ALLOCATED".equals(type) && dueDate != null) {
            try {
                UserClient.UserProfileSummary profile = fetchUserProfileOrNull(userId);
                if (profile == null || profile.email() == null || profile.email().isBlank()) {
                    log.warn("Skipping borrow confirmation for borrowId {}: no email for userId {}", borrowId, userId);
                    return;
                }
                String displayName = profile.name() != null && !profile.name().isBlank() ? profile.name() : "Library user";
                notificationClient.sendBorrowConfirmation(new NotificationClient.BorrowConfirmationRequest(
                        profile.email(),
                        displayName,
                        borrowId,
                        bookId,
                        dueDate
                ));
